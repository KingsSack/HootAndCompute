A `ManualMode` is a teleoperated mode that allows you to control your `Robot` using gamepads.

## Guide

This guide will walk you through the process of setting up your own `ManualMode`.

A `ManualMode` links inputs (buttons, analog sticks, etc.) to `Action`s from your `Robot`.

### 1. Create Your `ManualMode` Class

First, create a new class that inherits `ManualMode`.
The constructor requires a `robotFactory`, which is used to create your `Robot` instance.
Use a lambda function to provide the `robotFactory`.

<Tabs>

<TabItem label="Kotlin">

```kotlin
import dev.kingssack.volt.opmode.manual.ManualMode

class ExampleTeleOp : ManualMode<MyRobot>(robotFactory = { hardwareMap -> MyRobot(hardwareMap) }) {
    init {
        // Your interactions will go here
    }
}
```

### 2. Map `Action`s to Inputs

The `ManualMode` class provides the following helper functions to map `Action`s to inputs:

1. `onButtonReleased(button: GamepadButton, block: VoltActionBuilder<R>.() -> Unit)`: Triggers when the button is
    released
2. `onButtonTapped(button: GamepadButton, block: VoltActionBuilder<R>.() -> Unit)`: Triggers when the button was
    tapped
3. `onButtonDoubleTapped(button: GamepadButton, block: VoltActionBuilder<R>.() -> Unit)`: Triggers when the button
    was double tapped
4. `onButtonHeld(button: GamepadButton, block: VoltActionBuilder<R>.() -> Unit)`: Triggers when the button was held
    for the specified time

The `ManualMode` class also provides the following continuous input functions:

1. `whileButtonHeld(button: GamepadButton, block: R.() -> Unit)`: Continuously triggers while the button is held
    down
2. `onAnalog(input: GamepadAnalogInput, block: R.(Float) -> Unit)`: Continuously triggers with the analog input value

Let's map "A" on gamepad one to score, "A" on gamepad two to open the claw, and "B" on gamepad two to close the
    claw.

<Tabs>

<TabItem label="Kotlin">

```kotlin {6-14}
import dev.kingssack.volt.opmode.manual.ManualMode
import dev.kingssack.volt.util.GamepadButton

class ExampleTeleOp : ManualMode<MyRobot>(robotFactory = { hardwareMap -> MyRobot(hardwareMap) }) {
    init {
        onButtonReleased(GamepadButton.A1) {
            +robot.score()
        }
        onButtonReleased(GamepadButton.A2) {
            +robot.claw.open()
        }
        onButtonReleased(GamepadButton.B2) {
            +robot.claw.close()
        }
    }
}
```

</TabItem>

<TabItem label="Java">

```java {8-16}
import dev.kingssack.volt.opmode.manual.ManualMode;
import dev.kingssack.volt.util.GamepadButton;

public class ExampleTeleOp extends ManualMode<MyRobot> {
    public ExampleTeleOp() {
        super(MyRobot::new);

        onButtonReleased(GamepadButton.A1, builder -> {
            builder.unaryPlus(robot.score());
        });
        onButtonReleased(GamepadButton.A2, builder -> {)
            builder.unaryPlus(robot.getClaw().open());
        });
        onButtonReleased(GamepadButton.B2, builder -> {
            builder.unaryPlus(robot.getClaw().close());
        });
    }
}
```

### 3. Make it an OpMode

Add the `@TeleOp` annotation to the class definition. Without this annotation, the OpMode will not be registered and will not appear on the Driver Station.

<Tabs>

<TabItem label="Kotlin">

```kotlin {5}
import com.qualcomm.robotcore.eventloop.opmode.TeleOp
import dev.kingssack.volt.opmode.manual.ManualMode
import dev.kingssack.volt.util.GamepadButton

@TeleOp(name = "Example TeleOp", group = "Examples")
class ExampleTeleOp : ManualMode<MyRobot>(robotFactory = { hardwareMap -> MyRobot(hardwareMap) }) {
    init {
        onButtonReleased(GamepadButton.A1) {
            +robot.score()
        }
        onButtonReleased(GamepadButton.A2) {
            +robot.claw.open()
        }
        onButtonReleased(GamepadButton.B2) {
            +robot.claw.close()
        }
    }
}
```

</TabItem>
