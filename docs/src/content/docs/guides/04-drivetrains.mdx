A `Drivetrain` is a type of [Attachment](../02-attachments) that defines [Actions](../../concepts/01-actions) that can move [Robots](../01-robots) autonomously and tele-operated.

## Steps

Volt provides many pre-built `Drivetrain` classes for many combinations of drivetrain types, localization methods, and backends. Using these pre-built classes is the simplest way to add a `Drivetrain` to your [Robot](../01-robots).

:::note

It is also possible to implement a custom `Drivetrain` class by extending one of the provided abstract classes. This guide assumes you will use one of the pre-built `Drivetrain` classes seen in the table below.

:::

:::tip[Prerequisites]

This guide assumes you've [created a Robot class](../01-robots), [created an Attachment class](../02-attachments), and [learned to use built-in Attachments](../03-built-in-attachments).

:::

### 1. Choose a `Drivetrain`

Choosing a `Drivetrain` is a three-step process:

1. Select a type of physical drivetrain (What drivetrain does your robot physically use?)
2. Select a localization method (What localization hardware do you have?)
3. Select a compatible backend (Which pathing backend/actions do you want to use?)

:::note[See Also]

See [RoadRunner vs PedroPathing](../../reference/drivetrain-backends) to decide which backend is best for you.

:::

Let's use a mecanum drivetrain, drive encoder localization, and PedroPathing.

### 2. Create a `DrivetrainRobot`

Make a `Robot` class that extends `DrivetrainRobot`. This gives your [Robot](../01-robots) access to a `drivetrain` instance.

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin "DrivetrainRobot"
class MyRobot(hardwareMap: HardwareMap) :
    DrivetrainRobot(hardwareMap)
```

</TabItem>

<TabItem label="Java">

```java "DrivetrainRobot"
public class MyRobot extends DrivetrainRobot {
    public MyRobot(HardwareMap hardwareMap) {
        super(hardwareMap);
    }
}
```

</TabItem>

</Tabs>

### 3. Declare `Drivetrain` Type

Pass a generic type parameter that inherits `Drivetrain`. This lets Volt know what type of `Drivetrain` class your [Robot](../01-robots) uses.

In this case we'll use `DriveEncoderMecanumPedroPathingDrivetrain`:

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin "<DriveEncoderMecanumPedroPathingDrivetrain>"
class MyRobot(hardwareMap: HardwareMap) :
    DrivetrainRobot<DriveEncoderMecanumPedroPathingDrivetrain>(hardwareMap)
```

</TabItem>

<TabItem label="Java">

```java "<DriveEncoderMecanumPedroPathingDrivetrain>"
public class MyRobot extends DrivetrainRobot<DriveEncoderMecanumPedroPathingDrivetrain> {
    public MyRobot(HardwareMap hardwareMap) {
        super(hardwareMap);
    }
}
```

</TabItem>

</Tabs>

### 4. Create and Configure `Drivetrain` Instance

Create a new instance of your `Drivetrain` type. This is where you define hardware and tune your `Drivetrain`.

:::tip

Learn how to tune your robot on the [Tuning Your Robot](../../reference/tuning) page. Constants vary with each `Drivetrain` implementation.

:::

For our `DriveEncoderMecanumDrivetrain`, we'll supply simple `FollowerConstants`, `DriveEncoderConstants`, and `MecanumConstants`.

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin {4-17}
class MyRobot(hardwareMap: HardwareMap) :
    DrivetrainRobot<DriveEncoderMecanumPedroPathingDrivetrain>(
        hardwareMap,
        DriveEncoderMecanumPedroPathingDrivetrain(
            hardwareMap = hardwareMap,
            followerConstants = FollowerConstants()
                .mass(9.8)
                .translationalPIDFCoefficients(PIDFCoefficients(0.02, 0.0, 0.0, 0.07)),
            localizerConstants = DriveEncoderConstants()
                .robotWidth(14.0)
                .robotLength(9.0)
                .forwardTicksToInches(0.0057725)
                .strafeTicksToInches(0.0069642)
                .turnTicksToInches(0.009961),
            driveConstants = MecanumConstants()
                .maxPower(0.8),
        ),
    )
```

</TabItem>

<TabItem label="Java">

```java {5-20}
public class MyRobot extends DrivetrainRobot<DriveEncoderMecanumPedroPathingDrivetrain> {
    public MyRobot(HardwareMap hardwareMap) {
        super(
            hardwareMap,
            new DriveEncoderMecanumPedroPathingDrivetrain(
                hardwareMap,
                new FollowerConstants()
                    .mass(9.8)
                    .translationalPIDFCoefficients(new PIDFCoefficients(0.02, 0.0, 0.0, 0.07)),
                new DriveEncoderConstants()
                    .robotWidth(14.0)
                    .robotLength(9.0)
                    .forwardTicksToInches(0.0057725)
                    .strafeTicksToInches(0.0069642)
                    .turnTicksToInches(0.009961),
                new PathConstraints(),
                new MecanumConstants()
                    .maxPower(0.8),
                new Pose()
            )
        );
    }
}
```

</TabItem>

</Tabs>

In this case, we would have calculated the [Robot](../01-robots)'s mass, driving base width, and driving base length; tuned translational PIDF coefficients, forward ticks per inch, strafe ticks per inch, and turn ticks per inch; and experimented with various maximum powers.

import com.pedropathing.control.PIDFCoefficients;

            )
        );
    }
}
