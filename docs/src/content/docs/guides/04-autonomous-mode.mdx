An `AutonomousMode` is a [VoltOpMode](../03-opmodes) that allows you to perform [Actions](../../concepts/01-actions) autonomously.

## Steps

:::tip[Prerequisites]

This guide assumes you've [created a Robot class](../01-robots), [created an Attachment class](../02-attachments), [learned about the structure of Volt OpModes](../03-opmodes), and understand 
[Actions](../../concepts/01-actions).

:::

### 1. Create Your `AutonomousMode` Class

First, create a new class that inherits `AutonomousMode`.

`AutonomousMode` takes a type parameter for your `Robot` class. It uses that type for the abstract `robot` property.

Override the `robot` property with an instance of your `Robot` class. You can use properties defined in `VoltOpMode` like `hardwareMap` to create your `Robot` instance.

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin
import dev.kingssack.volt.opmode.autonomous.AutonomousMode

class ExampleAuto : AutonomousMode<MyRobot>() {
    override val robot = MyRobot(hardwareMap)
}
```

</TabItem>

<TabItem label="Java">

```java
import dev.kingssack.volt.opmode.autonomous.AutonomousMode;

public class ExampleAuto extends AutonomousMode<MyRobot> {
    @NotNull MyRobot robot = new MyRobot(getHardwareMap());

    @Override
    protected @NotNull MyRobot getRobot() {
        return robot;
    }

    public ExampleAuto() {
        super();
    }
}
```

</TabItem>

</Tabs>

### 2. Define an Event

By default `AutonomousMode`s only have one event: `Start`, which is triggered when the play button is pressed on the Driver Station.

Events are defined with the `then` function, which takes an event and a lambda with a [VoltActionBuilder](../../concepts/03-volt-action-builder) as the receiver.

On `Start`, we'll open the claw, then close the claw and score at the same time. We'll repeat this sequence three times:

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin {7-18}
import dev.kingssack.volt.opmode.autonomous.AutonomousMode
import dev.kingssack.volt.util.Event.AutonomousEvent.Start

class ExampleAuto : AutonomousMode<MyRobot>() {
    override val robot = MyRobot(hardwareMap)

    init {
        Start then {
            repeat(3)  {
                +robot.claw.open() // Add an action to the sequence
    
                parallel { // Run these actions at the same time
                    +robot.claw.close()
                    +robot.score()
                }
            }
        }
    }
}
```

</TabItem>

<TabItem label="Java">

```java {16-26}
import dev.kingssack.volt.opmode.autonomous.AutonomousMode;
import dev.kingssack.volt.util.Event.AutonomousEvent.Start;
import kotlin.Unit;

public class ExampleAuto extends AutonomousMode<MyRobot> {
    @NotNull MyRobot robot = new MyRobot(getHardwareMap());

    @Override
    protected @NotNull MyRobot getRobot() {
        return robot;
    }

    public ExampleAuto() {
        super();

        then(Start.INSTANCE, builder -> {
            for (int i = 0; i < 3; i++) {
                builder.unaryPlus(robot.getClaw().open()); // Add an action to the sequence

                builder.parallel(() -> { // Run these actions at the same time
                    builder.unaryPlus(robot.getClaw().close());
                    builder.unaryPlus(robot.score());
                });
            }
            return Unit.INSTANCE;
        });
    }
}
```

</TabItem>

### 3. Add Metadata

Add the `@VoltOpModeMeta` annotation to the class definition. Without this annotation, the OpMode will not be registered and will not appear on the Driver Station.

<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin {5}
import dev.kingssack.volt.opmode.VoltOpModeMeta
import dev.kingssack.volt.opmode.autonomous.AutonomousMode
import dev.kingssack.volt.util.Event.AutonomousEvent.Start

@VoltOpModeMeta("Example Auto", "Examples")
class ExampleAuto : AutonomousMode<MyRobot>() {
    override val robot = MyRobot(hardwareMap)

    init {
        Start then {
            repeat(3) {
                +robot.claw.open()
        
                parallel {
                    +robot.claw.close()
                    +robot.score()
                }
            }
        }
    }
}
```

</TabItem>

## MultiAutonomousMode and DualAutonomousMode

Sometimes you may need multiple variants of an AutonomousMode, such as if your path needs to be mirrored for one color, or it can start in multiple locations.
That is the reason for the MultiAutonomousMode, DualAutonomousMode, and MultiDualAutonomousMode are for.
the MultiAutonomousMode allows you to create multiple instances of an autonomous for each instance of an enum, the DualAutonomousMode does the same thing specifically for team colors, and the MultiDualAutonomousMode is a combination of both.


<Tabs syncKey="language">

<TabItem label="Kotlin">

```kotlin {7-16}
import dev.kingssack.volt.opmode.autonomous.AutonomousMode
import dev.kingssack.volt.util.Event.AutonomousEvent.Start

class ExampleAuto : AutonomousMode<MyRobot>() {
    override val robot = MyRobot(hardwareMap)
    init {
        Start then {
                repeat(3)  {
                    +robot.claw.open() // Add an action to the sequence

                    parallel { // Run these actions at the same time
                        +robot.claw.close()
                        +robot.score()
                    }
                }
            }
    }
}
```

</TabItem>

<TabItem label="Java">

```java {17-27}
import dev.kingssack.volt.opmode.autonomous.MultiDualAutonomousMode;
import dev.kingssack.volt.opmode.autonomous.DualAutonomousMode.AllianceColor
import org.jetbrains.annotations.NotNull;

public class ExampleAuto extends MultiDualAutonomousMode<MyRobot, type> {
    enum type {
        type1,
        type2
    }
    @NotNull MyRobot robot = new MyRobot(getHardwareMap());

    @Override
    protected @NotNull MyRobot getRobot() {
        return robot;
        // example: check the type
        if (variant === type.type1) {}
        // example: check the color
        if (color === AllianceColor.RED) {}
        // helper methods:
        // mirroredForAlliance(pose) mirrors a pose if the blue OpMode is active
        // forAlliance(red, blue) returns a different value depending on if the red or blue OpMode is active
    }

    public ExampleAuto() {
        super();
    }
}
```
